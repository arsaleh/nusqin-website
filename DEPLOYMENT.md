# NuSQIN Website Deployment Guide

This document provides instructions for deploying the NuSQIN Medical Aesthetics website to Vercel or Railway.

## üöÄ Quick Deploy

### Vercel (Recommended for Next.js)

#### Option 1: Deploy via Vercel Dashboard
1. Go to [vercel.com](https://vercel.com)
2. Click "New Project"
3. Import your Git repository
4. Vercel will auto-detect Next.js settings
5. Click "Deploy"

#### Option 2: Deploy via Vercel CLI
```bash
# Install Vercel CLI
npm i -g vercel

# Deploy from project directory
cd /Users/arsl/code/beauty-clinic/nusqin-website
vercel

# Follow the prompts:
# - Set up and deploy? Yes
# - Which scope? (Select your account)
# - Link to existing project? No
# - Project name? nusqin-website
# - Directory? ./
# - Override settings? No
```

### Railway

#### Deploy via Railway Dashboard
1. Go to [railway.app](https://railway.app)
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Choose your repository
5. Railway will auto-detect Next.js
6. Click "Deploy"

#### Deploy via Railway CLI
```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Initialize and deploy
cd /Users/arsl/code/beauty-clinic/nusqin-website
railway init
railway up
```

## üåê Custom Domain Setup

### Vercel
1. Go to Project Settings ‚Üí Domains
2. Add your domain (e.g., `nusqin.com`)
3. Update DNS records as instructed
4. Wait for DNS propagation (up to 48 hours)

### Railway
1. Go to Project Settings
2. Click "Generate Domain" for Railway subdomain
3. Or add custom domain in "Custom Domains" section
4. Update DNS records as instructed

## üÜò Troubleshooting

### Build Failures

If build fails, check:
1. `npm run build` works locally
2. All dependencies in `package.json`
3. No TypeScript errors
4. Image URLs are accessible

### Images Not Loading

If external images (nusqin.com) don't load:
1. Check `next.config.ts` has correct `remotePatterns`
2. Verify image URLs are still valid
3. Check browser console for errors

### Form Not Submitting

If HubSpot form doesn't work:
1. Verify Portal ID and Form ID in `lib/constants.ts`
2. Check browser console for errors
3. Test form directly on HubSpot
